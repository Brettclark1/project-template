# docs/liability-reviews/

**Purpose:** Home for founder attestation reports — the documents Ryven files and keeps as the official record of what was delivered, what was reviewed, what was verified vs. assumed, and what the founder is prepared to explain on the record.

**Who writes here:** `liability-review` skill — before any client delivery. One report per delivery, named `{client}-{date}.md`.

**Who reads here:** Brett (founder attestation), clients during audits, and Ryven itself 18 months later when a dispute or compliance request arrives.

**Liability value:** this is the document you keep. If a client disputes something later, this is your record.

**Do not delete files in this directory.** Every report is a permanent liability instrument. Supersede via a new dated report, never overwrite.

See `ryven-brain/doctrine/ryven-build-certification.md` for the full framework.
